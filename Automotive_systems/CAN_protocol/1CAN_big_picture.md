# CAN Protocol – Overview

## Why CAN Exists
Modern vehicles contain multiple Electronic Control Units (ECUs) such as:
- Engine ECU
- ABS ECU
- Airbag ECU
- Transmission ECU

These systems must communicate reliably in real time.

Using direct point-to-point wiring would create excessive wiring complexity in vehicles.

To solve this problem, Controller Area Network (CAN) was introduced.

## Key Features
- Shared communication bus
- Reduced wiring complexity
- Real-time communication
- High reliability
- Differential signaling using CAN_H and CAN_L

## Developed By
Robert Bosch GmbH (1980s)

## Automotive Relevance
CAN protocol is widely used for communication between ECUs in modern vehicles.

## Key Learning
CAN significantly reduces wiring complexity while enabling reliable communication between embedded automotive systems.
